# JobLedger Candidate Packet

Build the `jobledger` package described in `prd.md`.

You are given a starter skeleton under `starter/`. The skeleton defines the
public module boundaries and function/class signatures. Preserve those public
interfaces and fill in the implementation. You may add private helpers and
additional modules, but hidden tests will import the documented public modules
and run the CLI.

Do not depend on network access or wall-clock time. Use explicit virtual time
arguments where provided.
